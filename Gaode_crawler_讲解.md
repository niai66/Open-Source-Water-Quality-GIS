## 二、前置知识：POI 是什么？高德 API 限制是什么？

### 什么是 POI？

POI = **Point of Interest**（兴趣点）。就是地图上的一个地点标记，比如：

- 一家叫"肯德基"的餐厅
- 一个叫"海航大厦"的写字楼
- 一个叫"海口市人民医院"的医院
- 一个叫"3路公交站"的公交站

每个 POI 都包含名称、地址、经纬度、电话、类型等信息。

### 高德 API 的限制

这个程序用的是高德地图的 **多边形搜索 API**（/v3/place/polygon），它的限制是：

| 限制项 | 具体数值 | 后果 |
|--------|----------|------|
| 每页最多返回 | 25 条 POI | 数据多时必须翻页 |
| 最多翻页数 | 约 36 页 | 一个区域最多返回 25x36 = 900 条 |
| 每日配额 | 个人 5000 次 / 企业 30 万次 | 用完就没了 |

**问题：** 海口市光"住宅小区"就有几千个，一个区域超过 900 条就会被截断，数据不全。

**这个程序的核心价值就是解决这个问题——用"动态网格切分"来绕过 900 条上限。**

## 三、核心思路：动态网格切分（最关键的设计）

### 直观理解

```
假设你要数清楚一个足球场上所有人（超过 900 人），但规则是：
你一次最多只能数一个格子里的 900 人，怎么办？

答案是：把足球场分成 4 块，每块少于 900 人就数，
如果某一块还超过 900 人，就继续分成更小的 4 块……
直到每一块都少于 900 人，再分别统计。
```

这就是这个程序的核心思想。

### 具体实现

```
初始网格 (单个矩形区域，如"龙华区核心")
    │
    ├─ 发请求问高德：这个区域有多少 POI？
    │
    ├─ 判断：数量 > 200（阈值）吗？
    │   ├─ 否 → 直接逐页采集全部数据（已安全）
    │   └─ 是 → 数量太多，一个区域装不下！
    │           ↓
    │           把这个网格四等分
    │           │
    │           ├─ 对每个子网格，重复上面的判断
    │           │   （递归，最多切 6 层）
    │           │
    │           └─ 直到所有子网格的 POI 数量都 <= 200
    │               或者网格已经小到不能再切（小于 0.01 度 ≈ 1km）
    │
    └─ 对每个不再切分的网格，逐页采集所有数据
```

### 停止切分的条件

```python
should_split = (
    depth < self.max_grid_depth and      # 条件1：还没到最大深度（6层）
    total_count > self.poi_threshold and # 条件2：POI 数量超过 200
    grid_width > self.min_grid_size and  # 条件3：网格宽度 > 0.01度
    grid_height > self.min_grid_size     # 条件4：网格高度 > 0.01度
)
```

四个条件必须全部满足才切分，任何一个不满足就直接采集。

### 4.4 API 密钥管理（第 170-223 行）

- **_validate_api_keys()**：启动时逐个测试密钥有效性，无效的自动剔除

每密钥每秒最多发 8 个请求（max_qps_per_key=8）。距离上次请求不足 1/8 秒就 sleep 等待，防止被封禁。

### 4.6 核心 API 请求 search_by_polygon（第 239-304 行）

程序唯一发 HTTP 请求的地方。

**构造多边形参数：** 将矩形 [min_lng, min_lat, max_lng, max_lat] 转换成坐标串：
- 左下角 -> 右下角 -> 右上角 -> 左上角 -> 回到左下角

**请求参数：**
```python
params = {
    "key": api_key,           # API 密钥
    "polygon": polygon,       # 搜索区域多边形
    "types": types,           # POI 类型编码
    "offset": 25,             # 每页 25 条
    "page": page,             # 页码
    "extensions": "all",      # 返回全部字段
    "output": "json"          # 返回 JSON 格式
}
```

**返回结果：**
- status=1 表示成功，返回 (pois列表, 总数)
- 遇到 DAILY_QUERY_OVER_LIMIT 标记密钥失效
- 其他错误自动重试（最多 3 次）

### 4.7 核心递归方法 collect_in_grid（第 347-438 行）

```
collect_in_grid(type_info, bounds, grid_name, depth=0)
    → 1. 检查网格是否已处理（去重）
    → 2. 查询网格内 POI 总数
    → 3. 判断：需要切分吗？
       ├─ 是 → 四等分，递归调用自己（depth+1）
       └─ 否 → 逐页采集全部数据
    → 4. 返回采集数量
```

**切分操作：** 计算中心点，分成西南/东南/西北/东北 4 个子网格
**采集操作：** 翻页循环，每页 25 条，返回不足 25 条时停止

### 4.8 按网格系统采集 collect_by_grid_system（第 440-476 行）

先在 4 个核心区域采集，如果总和不足 100 条，就用全市矩形兜底采集。

### 4.9 断点续采：检查点机制（第 478-503 行）

- **load_checkpoint()**：启动时读取 checkpoint Excel，恢复已采 POI_ID 到去重集合

### 4.10 全流程调度 run_full_collection（第 505-537 行）

## 七、常见问题 FAQ

**Q1: 这个脚本我能直接用吗？**
A: 需要高德地图 API 密钥，填到 main() 函数的 API_KEYS 列表中。

**Q2: 一个密钥够用吗？**
A: 企业密钥日配额约 30 万次，够用。个人密钥日配额 5000 次，建议准备 3-5 个。

**Q3: 采集一次要多久？**
A: 海口市约 2-5 万条 POI，预计 1-3 小时。

**Q4: 采集到一半断了怎么办？**
A: 重新运行脚本，会自动从 checkpoint 文件恢复进度继续采集。

**Q5: 可以采集其他城市吗？**
A: 可以，修改 city_bounds 和 grid_bounds 的经纬度范围即可。

**Q6: 为什么我用不了？**
A: 常见原因：① API 密钥无效 ② 每日配额用尽 ③ 高德服务限制。检查日志文件。
